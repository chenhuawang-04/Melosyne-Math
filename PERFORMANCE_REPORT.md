# MMath vs GLM 性能基准测试报告

## 测试环境
- **编译器**: Clang 21.1.5
- **优化标志**: `-O3 -march=native -ffast-math -funroll-loops -finline-functions`
- **操作系统**: Windows 10
- **平台**: x64

## 基准测试结果

### 向量运算

#### Vec2 加法 (1M操作)
```
MMath:  0.914 ms
GLM:    0.902 ms
比率:   0.988x (GLM略快)
```
- **结论**: 两者性能相当，都经过高度优化

#### Vec2 点积 (1M操作)
```
MMath:  0.760 ms
GLM:    0.775 ms
比率:   1.019x (MMath快)
```
- **结论**: MMath 稍微快 1.9%

#### Vec2 规范化 (1M操作)
```
MMath:  0.644 ms
GLM:    0.679 ms
比率:   1.054x (MMath快)
```
- **结论**: MMath 快 5.4%

### 矩阵运算

#### 3x3 矩阵乘法 (100K操作)
```
MMath:  0.486 ms
GLM:    0.655 ms
比率:   1.347x (MMath快)
```
- **结论**: **MMath快34.7%** - 由于优化的仿射乘法

#### 矩阵变换点 (1M操作)
```
MMath:  1.992 ms
GLM:    1.954 ms
比率:   0.981x (GLM略快)
```
- **结论**: 性能相当，差异在误差范围内

### 包围盒运算（AABB2 - GLM不支持）

#### AABB2 交集测试 (10M操作)
```
MMath:  14.540 ms (per 10M ops)
GLM:    不支持内置AABB2
比率:   N/A
```

#### AABB2 合并 (10M操作)
```
MMath:  13.810 ms (per 10M ops)
GLM:    不支持内置AABB2合并
比率:   N/A
```

**优势**: MMath 提供专门的AABB2操作，通过单条SSE指令实现合并

### 三角函数运算

#### sin/cos (1M操作)
```
MMath:  0.489 ms
GLM:    13.380 ms
比率:   27.374x (MMath快)
```
- **结论**: **MMath快27倍** - 使用手写SIMD + DirectXMath多项式近似

## 性能总结

| 运算类别 | MMath优势 | 说明 |
|---------|----------|------|
| Vec2加法 | 相当 | 两者都经过高度优化 |
| Vec2点积 | +1.9% | 轻微优势 |
| Vec2规范化 | +5.4% | 合理优势 |
| Mat3乘法 | **+34.7%** | 优化的仿射乘法 |
| Mat3变换 | 相当 | 性能相当 |
| **三角函数** | **+2637%** | 手写SIMD实现 |
| AABB2操作 | 独有功能 | GLM无AABB2支持 |

## 关键发现

### MMath的优势
1. **专门优化的2D运算** - 所有函数针对2D用例优化
2. **手写SIMD代码** - AVX2/SSE4.1 手工编写的内部循环
3. **AABB2支持** - GLM不提供2D包围盒操作
4. **三角函数性能** - 使用DirectXMath多项式+SIMD，性能超GLM 27倍

### GLM的优势
1. **通用库** - 支持2D/3D/4D所有维度
2. **广泛应用** - 成熟、广泛使用的标准库
3. **跨平台** - 更广泛的平台和编译器支持

## 建议

### 使用 MMath 的场景
- 2D游戏引擎
- UI渲染系统
- 2D图形处理
- 实时性能关键的应用（特别是三角函数密集型）
- 需要AABB2操作的应用

### 使用 GLM 的场景
- 需要3D数学库的应用
- 跨平台应用（需要广泛兼容性）
- 不需要2D特化优化的应用

## 编译优化
所有基准测试都使用了积极的优化标志：
```bash
-O3 -march=native -ffast-math -funroll-loops -finline-functions
```

这些标志对MMath和GLM都有利，展示了两个库在最优条件下的性能。
