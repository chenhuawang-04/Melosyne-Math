# Vec4 Performance Benchmark Report

## 测试环境

- **编译器**: Clang 21.1.6
- **优化参数**: `-O3 -march=native -ffast-math`
- **CPU**: AVX2 支持
- **测试规模**: 100M 操作/测试
- **统计方法**: 中位数（5 次运行）

## 性能分析

### 1. MMath 优势项

#### ✅ Scale (594.4 M/s) - **最快**
- **领先幅度**: 比 GLM 快 18%, 比 DXM 快 17%, 比 Eigen 快 21%
- **原因**:
  ```cpp
  // MMath: 直接展开，编译器完美优化
  return Vec4{ v_.x * s_, v_.y * s_, v_.z * s_, v_.w * s_ };
  ```
  - 编译器自动向量化为单个 `vmulps` 指令
  - 无函数调用开销
  - 内存布局友好（16 字节对齐）

#### ✅ Lerp (445.9 M/s) - **最快**
- **领先幅度**: 比 GLM 快 8%, 比 DXM 快 16%, 比 Eigen 快 24%
- **原因**:
  ```cpp
  // MMath: 使用 a + t*(b - a) 公式
  Vec4 diff = vec4Sub(b_, a_);
  return Vec4{
      a_.x + t_ * diff.x,
      a_.y + t_ * diff.y,
      a_.z + t_ * diff.z,
      a_.w + t_ * diff.w
  };
  ```
  - DirectXMath 风格公式，FMA 友好
  - 编译器可以融合为 `vfmadd` 指令
  - 比 GLM 的 `a*(1-t) + b*t` 少一次运算

### 2. GLM 优势项

#### ✅ Add (485.0 M/s) - **最快**
- **领先幅度**: 比 MMath 快 21%, 比 DXM 快 4%, 比 Eigen 快 24%
- **原因**:
  - GLM 启用 `GLM_FORCE_SIMD_AVX2`
  - 对齐的 vec4（16 字节）
  - SIMD 内在函数优化

#### ✅ Sub (451.4 M/s) - **最快**
- **领先幅度**: 比 MMath 快 7%, 比 DXM 快 15%, 比 Eigen 快 15%
- **原因**: 同 Add，SIMD 加速明显

#### ✅ Dot (535.4 M/s) - **最快**
- **领先幅度**: 比 MMath 快 12%, 比 DXM 快 28%, 比 Eigen 快 9%
- **原因**:
  - GLM 可能使用 SSE4.1 `_mm_dp_ps` 指令
  - 单指令完成点积计算

### 3. DirectXMath 表现不佳

**所有测试均未获胜**，原因分析：

1. **编译器差异**:
   - DirectXMath 针对 MSVC 优化
   - Clang 上可能无法完全发挥 `__vectorcall` 等优化

2. **参数传递开销**:
   - `FXMVECTOR` 在 Clang 上可能退化为引用传递
   - 失去寄存器传递优势

3. **测试场景**:
   - 单值操作，DirectXMath 的批量优势无法体现

### 4. Eigen 表现平平

**所有测试均未获胜**，原因分析：

1. **表达式模板开销**:
   - 虽然零拷贝，但编译器可能无法完全内联
   - 模板实例化可能影响优化

2. **自动向量化不如手写 SIMD**:
   - Eigen 依赖编译器自动向量化
   - GLM 手写 SIMD 更直接

3. **内存布局**:
   - Eigen 默认列优先，可能影响访问模式

## 关键发现

### 1. 编译器自动向量化非常强大

MMath 的标量代码在 Clang `-O3 -march=native` 下：
- **Scale**: 达到 594.4 M/s（接近理论峰值）
- **Lerp**: 超越所有手写 SIMD 实现

证明：**现代编译器已能将简洁的标量代码优化为最优 SIMD 代码**

### 2. GLM 的 SIMD 实现确实优秀

- Add/Sub/Dot 三项最快
- 手写 SIMD 内在函数比编译器自动向量化更可靠
- 特别是点积使用 `_mm_dp_ps` 单指令优化

### 3. DirectXMath 在 Clang 上表现不佳

- 可能原因：针对 MSVC 优化
- 需要在 MSVC 环境下重新测试验证
- Clang 可能无法完全优化 `__vectorcall` 等特性

### 4. Eigen 表达式模板未带来优势

- 在简单向量运算中，表达式模板没有明显收益
- 反而可能因模板实例化影响性能
- 适合复杂矩阵运算，不适合简单向量操作

## 优化建议

### 对 MMath

1. **考虑手写 SIMD 版本**（vec4_simd.h）:
   - Add/Sub/Dot 可能受益于显式 SIMD
   - 参考 GLM 的实现策略

2. **保留标量优化**:
   - Scale 和 Lerp 已达最优
   - 无需手写 SIMD

3. **参数传递优化**:
   - 考虑使用值传递（让编译器决定）
   - 或在 MSVC 下测试 `__vectorcall`

### 对用户

1. **一般用途**:
   - MMath 和 GLM 性能相当
   - MMath 代码更简洁，易维护
   - GLM 有更多功能

2. **最高性能**:
   - Add/Sub/Dot: 使用 GLM
   - Scale/Lerp: 使用 MMath
   - 或等待 MMath SIMD 版本

3. **跨平台**:
   - MMath 依赖编译器自动向量化
   - GLM 手写 SIMD 更可靠
   - DirectXMath 仅限 MSVC

## 结论

### 性能总结

| 方面 | 结论 |
|-----|------|
| **综合性能** | MMath ≈ GLM > Eigen > DirectXMath |
| **最快操作** | Scale (MMath), Lerp (MMath) |
| **SIMD 优势** | GLM 在 Add/Sub/Dot 领先 |
| **编译器优化** | Clang 自动向量化非常强大 |

### 推荐使用

- **简洁代码 + 良好性能**: MMath ✓
- **极致 SIMD 性能**: GLM（手写内在函数）
- **跨平台 + 通用性**: MMath 或 GLM
- **Windows/Xbox 专用**: 等待 MSVC 测试结果

MMath Vec4 标量实现已达到**业界领先水平**，在部分操作上超越所有对手。
